# Refactor Print Station UI and ZPL Template for Enterprise MES

## Objective

Refactor the current Print Station implementation to align with a real Manufacturing Execution System (MES).

The current implementation is only a proof of concept.

Current generated ZPL:

```zpl
^XA
^FO50,50^A0N,36,36^FDJob: evt-sim-0e45ce528344435da489f65eba6e91dd^FS
^FO50,100^A0N,36,36^FDSKU: PO-2026-0001^FS
^FO50,150^A0N,36,36^FDSerial: PO-2026-0001^FS
^XZ
```

This template is unacceptable for a production MES because almost all manufacturing information is missing.

The Print Station must become a complete manufacturing workstation instead of only sending three text fields to Zebra.

---

# Manufacturing Background

The company manufactures industrial rubber products.

Examples:

- Bearing Seals
- O-Rings
- Rubber Gaskets
- Rubber-to-Metal Bonding Parts
- Automotive Rubber Components

Printing is only ONE operation inside the workflow.

Workflow example:

Mixing
↓

Extrusion
↓

Cutting
↓

Vulcanization
↓

Cleaning
↓

Visual Inspection
↓

Printing Label
↓

Packaging

Therefore the Print Station only receives Work Orders generated by MES.

# JSON Template

Label template should no longer be hardcoded.

Store templates as JSON.

Example

{
    "paperWidth":100,
    "paperHeight":60,
    "elements":[
        {
            "type":"text",
            "field":"product_name"
        },
        {
            "type":"datamatrix",
            "field":"trace_id"
        }
    ]
}

Renderer converts

JSON

↓

ZPL

↓

Printer
