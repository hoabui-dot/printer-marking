# Feature Request: Dynamic Label Template Management & Printer Assignment
## Project: Print Marking Station (Station Agent)
## Priority: High
## Goal
Implement a complete enterprise-grade Label Template Management module inside the Station Agent so that operators and administrators can manage multiple label layouts, automatically adapt templates to different label sizes, preview labels before printing, and assign different templates to different printers without modifying source code.

This feature must integrate seamlessly with the existing MES → Job Engine → RabbitMQ → Station Agent architecture.

---

# Background

The current implementation hardcodes a single ZPL layout.

This is not suitable for manufacturing because:

- Different customers require different label layouts.
- Different products require different information.
- Different Zebra printers may use different label sizes.
- Different production lines may print different templates.
- Engineers currently need to edit ZPL manually whenever the label changes.

Instead, the Station Agent should become template-driven.

# Enterprise Requirements

- Maintain Database-per-Service architecture.
- Do not query another service's database directly.
- Use events for synchronization.
- Store template metadata inside the Station Agent database.
- Generate ZPL only inside the Station Agent.
- Make the template engine extensible for future barcode formats and printer models.
- Ensure backward compatibility so historical jobs always use the template version originally printed.
