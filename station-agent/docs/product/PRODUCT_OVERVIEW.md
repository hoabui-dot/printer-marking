## What Is This System?

The **Print-Marking Edge Station** is an **Industrial Edge Computing Platform** deployed directly on the factory production floor, adjacent to production machines.

It acts as the **local intelligence layer** between the Factory Gateway (cloud/central system) and physical production devices.

## Main Purpose

The station executes **print and/or marking operations** on manufactured products, ensuring every product is correctly identified, labeled, and verified.

### 8-Step Execution Loop

```
1. RECEIVE     вҶ’ Receive production event from Factory Gateway via MQTT
2. CREATE      вҶ’ Create a Job with all required parameters
3. GENERATE    вҶ’ Generate print/mark content (ZPL, serial numbers, QR)
4. EXECUTE PRINT  вҶ’ Send content to Label Printer
5. EXECUTE MARK   вҶ’ Send content to Laser Marking Machine
6. VERIFY      вҶ’ Confirm result via Vision/OCR System
7. PERSIST     вҶ’ Save result to local SQLite database
8. SYNC        вҶ’ Synchronize result back to Factory Gateway via MQTT
```

Not all steps are required for every operation. See [MANUFACTURING_WORKFLOW.md](./MANUFACTURING_WORKFLOW.md) for the exact flows.

## Core Business Domains

### 1. Product Identification

Every manufactured product must receive a unique identifier generated by the station.

**Format Examples:**
```
FC-WP-RO100G-B-998822       вҶ’ Factory Code + Product + Serial
FC-SHAMPOO-250ML-000112     вҶ’ Factory Code + SKU + Sequence
FC-LOTION-500ML-883722      вҶ’ Factory Code + SKU + Serial
```

**Rules:**
- Identifiers are globally unique within the production batch
- Format is defined by the production recipe from Factory Gateway
- Station must not generate duplicate identifiers

---

### 2. Label Printing

The station sends print commands to label printers attached to the production line.

**Content types that can be printed:**
| Content Type | Example |
|---|---|
| QR Code | Product URL or traceability code |
| Barcode | GS1 128, Code 39, EAN-13 |
| Text | Lot number, expiry date, SKU |
| Lot Number | LOT-2026-06-A-001 |
| Manufacturing Date | 2026-06-16 |
| Expiry Date | 2028-06-16 |

**Protocol:** TCP port 9100, ZPL or EPL command language.

---

### 3. Laser Marking

The station sends marking commands to laser machines that engrave information directly onto product packaging or the product itself.

**Content types that can be marked:**
| Content Type | Example |
|---|---|
| Lot Number | 2026-BATCH-A |
| Date Code | 260616 |
| Serial Number | SN-0001234 |
| Traceability Code | TRC-FC-WP-RO100G-998822 |
| QR Code | Laser-etched QR |
| Barcode | Dot-peen barcode |

**Marking methods:** LASER_ETCHING, LASER_DOT_PEEN, LASER_SERIALIZATION, LASER_QR_MARKING, LASER_BARCODE_MARKING

---

### 4. Verification

After printing/marking, the station verifies the output using the Vision System.

**Verification methods:**
| Method | Tool | Use Case |
|---|---|---|
| OCR | Camera + OCR engine | Text readability check |
| Barcode Scan | Scanner | Barcode decodability |
| QR Scan | Camera | QR code integrity |
| Vision Inspection | Camera + AI | Label placement, defect detection |

**Verification results:** VERIFIED_PASS, VERIFIED_FAIL, VERIFIED_RETRY, VERIFIED_BYPASS

---

### 5. Traceability

Every operation performed on every product must be permanently recorded.

**What is recorded:**
- Job creation time and parameters
- Each execution step with timestamp
- Device used
- Result (pass/fail)
- Operator who approved any overwrite
- All retries and their outcomes

**Why:** Regulatory compliance, quality audits, recall investigations.

## Service Roles (Quick Reference)

| Service | Role |
|---|---|
| **MQTT Adapter** | Gateway between Factory MQTT and internal event bus |
| **Job Engine** | Core orchestration: creates, schedules, and tracks jobs |
| **Printer Adapter** | Communicates with label printers (ZPL/TCP) |
| **Laser Adapter** | Communicates with laser marking machines |
| **Vision Service** | Communicates with cameras/OCR for verification |
| **PLC Adapter** | Reads PLC state (triggers, machine status, line state) |
| **Kiosk UI** | Operator interface for monitoring and overrides |

## Key Business Rules

1. **No production without a job** вҖ” devices must not be commanded directly; all operations go through the Job Engine.
2. **All events must be logged** вҖ” no silent operations allowed.
3. **Failed verification must block the line** вҖ” unless explicitly bypassed by an authorized operator.
4. **Every sync to Gateway must be acknowledged** вҖ” retries are mandatory on failure.
5. **Overwrite operations require operator approval** вҖ” REPRINT, RELASER, FORCE_PASS, FORCE_COMPLETE all require a logged approval.
